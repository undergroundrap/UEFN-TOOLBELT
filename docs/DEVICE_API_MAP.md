# Fortnite Device API Map

This document maps the Python API for common Fortnite devices and level actors, derived from automated introspection of the UEFN level.

> [!TIP]
> Use `tb.run("api_crawl_level_classes")` to update this data for your specific level.

## Common Device & Actor Classes

| Class | Description | Key Methods / Properties |
| :--- | :--- | :--- |
| `FortCreativeLockDevice` | Lock Device | `toggle_active()`, `set_owner_player()`, `allow_interact`, `is_locked` |
| `BuildingDeco` | Prop / Decoration | `actor_has_tag()`, `destroy_actor()`, `static_mesh` |
| `BuildingProp` | Standard Prop | `set_health()`, `set_is_indestructible()`, `get_transform()` |
| `FortGameplayReceiverMessageComponent` | Message Receiver | `on_gameplay_message_received()`, `set_channel_id()`, `filter_tags` |
| `FortLocalizableMessageComponent` | Localized Text | `localize_and_set_message()`, `set_active()` |
| `BaseBuildingStaticMeshComponent` | Visual Mesh | `set_static_mesh()`, `set_material()`, `cast_shadow` |
| `Actor` | Base Class | `get_actor_location()`, `set_actor_rotation()`, `tags` |

## Technical Details

The full schema is available in your local saved directory after running the crawler:
`%LOCALAPPDATA%\UnrealEditorFortnite\Saved\UEFN_Toolbelt\api_level_classes_schema.json`

### Example Usage

```python
import UEFN_Toolbelt as tb
# Find a lock device and lock it
for actor in tb.get_selected_actors():
    if "LockDevice" in str(actor.get_class()):
        actor.call_method("toggle_active", [False])
```

## 🧠 Verse Schema Intelligence (Phase 14)

As of the Phase 14 update, the Toolbelt now features **"Verse Schema IQ"**. Unlike the level crawler (which only sees actors currently in your scene), the Schema IQ parses the project's `.digest.verse` files to understand every class in the project.

### Get Schema for Any Class
Use `api_verse_get_schema(class_name)` to retrieve the full property, event, and function map for any Verse device:

```python
import UEFN_Toolbelt as tb
schema = tb.run("api_verse_get_schema", class_name="my_verse_device")
# Returns: { 'properties': {...}, 'events': [...], 'functions': [...] }
```

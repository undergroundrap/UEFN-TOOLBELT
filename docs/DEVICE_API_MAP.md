> **Source of truth**: `docs/uefn_reference_schema.json` — 1.6 MB schema generated by
> running `api_crawl_level_classes` on a live UEFN level (76 actors, 14 unique classes).
>

## Quick Reference — Class Hierarchy

```
Actor  (base — all classes inherit these props)
├── FortStaticMeshActor          — static mesh, no gameplay
├── BuildingProp                 — destructible prop (inherits FortActor)
│   └── BuildingFloor            — floor/wall/ramp tile (structural)
│       └── FortCreativeDeviceProp — Creative device shell
├── FortPlayerStartCreative      — spawn point
├── FortMinigameSettingsBuilding — island/game settings manager
├── FortInspectorCameraCreative  — editor inspect camera
├── FortWaterBodyActor           — ocean / water volume
├── TextRenderActor              — 3D text in world
├── LevelBounds                  — world boundary box
├── WaterZone                    — water gameplay zone
├── WorldDataLayers              — streaming layer manager
└── WorldPartitionMiniMap        — minimap actor
```

## Universal Properties — `Actor` (base class)

Every actor in UEFN inherits these. These are the safest properties to read/write
across any class without first checking the type.

| Property | Type | Default | Automation Use |
|:---|:---|:---|:---|
| `hidden` | `bool` | `False` | Toggle visibility from Python |
| `tags` | `Array` | `None` | Read gameplay tags (use `actor_has_tag()`) |
| `actor_guid` | `Guid` | `None` | Unique identity across sessions |
| `actor_instance_guid` | `Guid` | `None` | Per-instance identity |
| `can_be_damaged` | `bool` | `True` | Invulnerability toggle |
| `replicates` | `bool` | varies | Whether actor syncs over network |
| `net_dormancy` | `NetDormancy` | `DORM_AWAKE` | Network optimization state |
| `is_spatially_loaded` | `bool` | `True` | World partition streaming |
| `root_component` | `SceneComponent` | `None` | Access transform via this component |
| `pivot_offset` | `Vector` | `{0,0,0}` | Visual offset from actor origin |
| `initial_life_span` | `float` | `0.0` | 0 = lives forever; >0 = auto-destroy |
| `spawn_collision_handling_method` | enum | `ALWAYS_SPAWN` | How spawning handles overlaps |
| `custom_time_dilation` | `float` | `1.0` | Local time scale (1.0 = normal) |
| `net_cull_distance_squared` | `float` | `225000000.0` | ~15,000 UU cull radius |
| `net_update_frequency` | `float` | `100.0` | Network updates per second |

**Restricted / Event properties** (readable via `dir()` but not `get_editor_property`):
- `on_actor_begin_overlap`, `on_actor_end_overlap`, `on_actor_hit`, `on_actor_touch`
- `_wrapper_meta_data` — internal UEFN wrapper, always raises on access

**Python pattern:**
```python
import UEFN_Toolbelt as tb
actors = unreal.EditorActorSubsystem().get_all_level_actors()
for a in actors:
    if not a.hidden:                        # read
        a.set_editor_property("hidden", True)   # write
```

`FortCreativeDeviceProp` and `BuildingFloor` both inherit from this.

```python
for a in unreal.EditorActorSubsystem().get_selected_level_actors():
```

## `FortCreativeDeviceProp` — Verse Device Shell

The base class for **all** Fortnite Creative devices (trigger volumes, item spawners,
team managers, etc.) when seen from the Python layer. The Verse class is resolved separately
via `.digest` files.

> **Key quirk**: `api_crawl_level_classes` sees these as `FortCreativeDeviceProp`.
> The actual Verse class is discovered via the "Named Auto-Link" pattern in `verse_schema.py`.

| Property | Type | Default | Notes |
|:---|:---|:---|:---|
| `surpress_health_bar` | `bool` | `True` | Devices hide the HP bar by default |
| `is_spatially_loaded` | `bool` | `False` | Devices persist outside streaming range |
| `can_export_navigation_collisions` | `bool` | `True` | Unlike plain props |
| `area_class` | `Any` | `None` | Always `None` — use Verse schema for class |
| `static_mesh` | `Any` | `None` | May return a mesh or `None` |
| All `BuildingProp` props | — | — | Fully inherited |

**How to read a device's Verse properties:**
```python
# Step 1: Get the schema for its Verse class
schema = tb.run("api_verse_get_schema", class_name="my_trigger_device")
# Returns: {"properties": {...}, "events": [...], "functions": [...]}

# Step 2: Read via getattr() — NOT get_editor_property()
val = getattr(actor, "is_enabled", None)
```

```python
for i, s in enumerate(sorted_starts):
    s.set_editor_property("priority_group", i)
```

**Find the island settings actor:**
```python
settings_actor = None
for a in unreal.EditorActorSubsystem().get_all_level_actors():
    if "MinigameSettings" in a.get_class().get_name():
        settings_actor = a
        break

# Read current player cap

# Set to 8-player
settings_actor.set_editor_property("max_players", 8)
```

## The 19 Restricted Properties

Every class has the same 19 restricted properties that **cannot** be read via
`get_editor_property()`. Access raises an exception. These are C++ delegates and
internal metadata that UEFN intentionally blocks from Python:

```
_wrapper_meta_data          — Internal UEFN Python wrapper
on_actor_begin_overlap      — C++ delegate
on_actor_end_overlap        — C++ delegate
on_actor_hit                — C++ delegate
on_actor_touch              — C++ delegate
on_destroyed                — C++ delegate
on_end_play                 — C++ delegate
on_input_touch_begin        — C++ delegate
on_input_touch_end          — C++ delegate
on_input_touch_leave        — C++ delegate
on_take_any_damage          — C++ delegate
on_take_point_damage        — C++ delegate
on_take_radial_damage       — C++ delegate
bnet_use_owner_relevancy    — duplicated via net_use_owner_relevancy
... (+ others not shown in current schema)
```

**Rule**: If a property read raises, use `getattr(actor, prop, None)` as a safe fallback.
For events, wire them in Verse — not Python.

## Property Access Cheatsheet

```python
# SAFEST: Works for all non-restricted editor properties
value = actor.get_editor_property("hidden")
actor.set_editor_property("hidden", False)

# FOR VERSE PROPS: getattr bypasses the editor property system
value = getattr(actor, "my_verse_prop", None)

# FOR COMPONENTS: Always go through the component handle
mesh_comp = actor.get_editor_property("static_mesh_component")
mesh = mesh_comp.get_editor_property("static_mesh") if mesh_comp else None

# TAGS: Use the GameplayTag helpers
tag = unreal.GameplayTag.request_gameplay_tag(unreal.Name("Game.Round.Active"))
container = actor.get_editor_property("tags")

# SAFE PROPERTY AUDIT: Never use inspect.getmembers() — it crashes on unreal objects
for name in dir(actor):
    try:
        val = getattr(actor, name)
    except Exception:
        pass
```

## Schema Intelligence — Verse Devices

The `uefn_reference_schema.json` only covers Python-visible C++ properties.
For Verse-scripted device properties (`@editable` vars, events, functions),
the schema is in `.digest.verse` files, parsed by `verse_schema.py`.

```python
# Get full schema for any Verse device by class name
schema = tb.run("api_verse_get_schema", class_name="my_game_manager")
# Returns: {
#   "status": "ok",
#   "class_name": "my_game_manager",
#   "schema": {
#     "properties": {"rounds": "int", "score_limit": "int", ...},
#     "events": ["OnRoundEnd", "OnPlayerEliminated"],
#     "functions": ["StartRound", "EndGame"]
#   }
# }

# Generate Verse declaration stubs from your level selection
tb.run("verse_gen_device_declarations")   # outputs @editable vars

# Refresh schema cache after adding new Verse files
tb.run("api_verse_refresh_schemas")
```
